# UI 调研：对标 Hermes 桌面端 / Codex / Claude 桌面端

> 目的：为 Sage Agent 的 run 态（会话）界面重设计提供依据。结论先行，再给对照与落地建议。
> 日期：2026-06-30。来源：用户提供的 Hermes 桌面端两张截图（空态 + 会话态）+ OpenAI Codex app 官方文档 + Anthropic「Claude Code 桌面端重设计」博客与 Claude Design 帮助文档。

## 结论

三个参考共享同一套设计语言：**「文档式会话」装在「双栏壳 + 富信息状态栏」里**。我们 Step 1–3 已经把空态首页和外壳对齐了；**剩下的差距集中在 run 态**——消息渲染、顶栏、composer、状态栏四块。它们恰恰是用户每天盯着看的部分。

## 共享设计语言（三者一致）

1. **双栏主布局**：常驻左侧「会话/会话列表」侧栏 + 大块主会话区。右侧面板一律按需（Claude 可拖拽分栏、Codex 弹出独立窗口、Hermes 极简不常驻）。
2. **会话中心化侧栏**：新建会话（带 ⌘N）、顶层导航入口、搜索、置顶区、带状态点的会话列表、底部 home/＋/更多。
3. **文档式消息渲染（非聊天气泡）**：
   - 用户消息 = 轻描边卡片（圆角、淡边框、几乎无填充），居于回合顶部。
   - 助手消息 = 纯文本散排、左对齐、**无头像、无「user/assistant」角色标签、无气泡**，读起来像文档。
   - 限定**阅读列宽**（居中、两侧留白），不铺满。
4. **极简顶栏**：只有会话标题 + 下拉 chevron（重命名/切换）+ 右上角工具图标（声音、快捷键、设置、面板切换）。没有重型 header、没有一排模型 chip。
5. **统一 composer 胶囊**：一个圆角容器内含 ＋（附件）、输入占位、模型选择器（如 `Agnes 2.0 Flash · Max ⌄`）、麦克风、圆形发送钮。运行时发送钮变停止。**不是** textarea + 下方一排按钮。
6. **富信息底部状态栏**：左侧 网关/代理/排程（Hermes）或 git/env（Codex/Claude）；右侧**上下文窗口计量**（`20.3k/256.0k ▓░ 8%`）、**会话计时**（`会话 0:25`）、版本号。这是这类「专业 agent 桌面」的标志性细节。
7. **气质**：大留白、克制、浅色、单一强调色、淡描边，常带一层极淡的背景纹理/插画（Hermes 有一匹隐约的马）。

## 各参考的差异化能力（供后续路线参考，非本期范围）

- **Codex app**：按项目分线程、worktree 隔离、线程弹出独立窗口、内联 diff 评审+评论、应用内浏览器/预览、语音输入（Ctrl+M）、线程内生成图片。
- **Claude 桌面端**：并行 session 侧栏（按状态/项目/环境过滤，PR 合并后自动归档）、side chat（⌘;）分叉提问、可拖拽分栏（终端/预览/diff/会话）、三个 tab（Chat/Cowork/Code）、三档视图（Verbose/Normal/Summary）。
- **Hermes**：整体最极简，把上面这些都收进了干净的双栏 + 文档式会话 + 富状态栏。

## 与我们当前实现（Step 1–3 后）的差距

| 区块 | 参考做法 | 我们现状 | 差距 |
| --- | --- | --- | --- |
| 顶栏 | 会话标题 + chevron + 右上图标 | 重型 header：标题 + 3 个模型 chip + launchpad 三卡 | 大：需删卡片、收成极简标题栏 |
| 消息 | 用户卡片 / 助手纯文本 / 无头像无标签 / 限宽 | 头像圆 + 角色标签 + article 气泡 / 满宽 | 大：需重写消息渲染 |
| composer | 统一胶囊 + 内联模型选择器 + 发送圆钮 | textarea + 下方按钮行 | 大：需重构 composer |
| 状态栏 | 上下文计量 + 会话计时 + 版本 | 仅 网关/排程/版本 | 中：缺计量与计时（依赖 run token 用量数据） |
| 侧栏 | 搜索 + 置顶 + 会话计数 + 导航入口 | settings + threads + runs | 中：缺搜索/置顶/导航入口 |

## 推荐的下一阶段（V2.1.2：run 态对齐）

按「最影响观感 → 次要」排序：

1. **消息文档化**：用户回合 = 描边卡片；助手 = 纯文本散排；去头像/去角色标签；正文限阅读列宽（约 720–820px 居中）。
2. **composer 胶囊化**：单容器 + ＋ + 输入 + 模型选择器 + 发送/停止圆钮；审批与取消改为会话内联或胶囊内状态。
3. **顶栏极简化**：删 launchpad 三卡、删模型 chip；顶部只留「会话标题 + chevron」+ 右上工具图标（含已做的 inspector 切换）。
4. **状态栏增强**：右侧加上下文窗口计量 + 会话计时。**依赖**：需确认 runtime 是否已暴露每个 run 的 token 用量与上下文上限；若无，先做静态/占位，待数据接入。

## 已确认决策（2026-06-30）

1. **消息渲染 = 文档式**：用户=描边卡片，助手=纯文本散排，去头像/去角色标签，限阅读列宽。
2. **composer = 统一胶囊**：单容器含 ＋/输入/模型选择器/发送圆钮；运行时发送钮变停止；取消=停止钮。
3. **本期范围 = 消息 + composer + 顶栏**：完整 run 态观感对齐（删 launchpad 与模型 chip，顶栏收成标题+chevron）。
4. **状态栏上下文计量延后**：已核实 runtime 当前无 token 用量埋点（grep 仅命中密钥脱敏/分词），需单独在 provider 层抓 DeepSeek `usage` 后才能做实。
5. **审批/inspector 暂留抽屉**：本期审批仍在 inspector，会话内联审批列为后续项（避免本轮范围膨胀）。
